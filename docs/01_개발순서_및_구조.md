# Vampire Survivors 클론 - 게임 개발 순서 & 구조

## 전체 개발 순서 (1인 개발 기준)

```
[1단계] 기획        →  무엇을 만들 것인가?
[2단계] 프로토타입   →  핵심 재미를 검증한다
[3단계] 본개발      →  시스템을 하나씩 완성한다
[4단계] 폴리싱      →  다듬고 버그를 잡는다
[5단계] 출시/배포    →  세상에 공개한다
```

---

## 1단계: 기획 (Planning)

> "어떤 게임을 만들 것인가?"를 글로 정리하는 단계

### 해야 할 일
- **게임 컨셉 정의**: 한 줄로 설명 가능해야 함
  - 예: "자동으로 공격하는 캐릭터로 몰려오는 몬스터를 쓸어버리는 로그라이크 액션"
- **핵심 메카닉(Core Mechanic) 정의**: 플레이어가 반복하는 행동
  - 예: "이동만 한다 → 자동 공격 → 경험치 획득 → 레벨업 → 무기 선택"
- **게임 루프(Game Loop) 설계**: 한 판의 흐름
  ```
  시작 → 캐릭터 선택 → 스테이지 진입 → 전투(이동+자동공격)
  → 경험치 획득 → 레벨업(무기/스킬 선택) → 보스 등장 → 사망 or 클리어
  → 결과화면 → 영구 업그레이드 → 다시 시작
  ```
- **필요한 리소스 목록 정리**: 캐릭터, 몬스터, 무기, UI, 사운드 등

### 산출물
- 기획서 (이 문서가 곧 기획서의 시작)
- 게임 루프 다이어그램
- 리소스 목록표

---

## 2단계: 프로토타입 (Prototype)

> 핵심 재미가 되는 부분만 최소한으로 만들어보는 단계

### 해야 할 일
- **개발 환경 세팅**: 엔진 설치, 프로젝트 생성
- **최소 기능 구현 (MVP)**:
  1. 캐릭터가 화면에서 움직인다 (상하좌우)
  2. 적이 캐릭터를 향해 다가온다
  3. 캐릭터가 자동으로 공격한다 (투사체 발사)
  4. 적이 죽으면 경험치(보석)를 떨어뜨린다
  5. 경험치를 모으면 레벨업한다
  6. 레벨업 시 무기/패시브 선택 UI가 뜬다

### 이 단계의 핵심
- **그래픽은 대충해도 됨** (사각형, 원 등 기본 도형으로 OK)
- **"이거 재미있나?"를 확인하는 것이 목적**
- 재미없으면 컨셉을 수정하고, 재미있으면 본개발로 진행

### 산출물
- 플레이 가능한 프로토타입 빌드

---

## 3단계: 본개발 (Production)

> 프로토타입을 기반으로 모든 시스템을 완성하는 단계

### 구현해야 할 시스템 (우선순위 순)

#### A. 코어 시스템 (Core Systems) - 게임의 뼈대
| 순서 | 시스템 | 설명 |
|------|--------|------|
| 1 | 게임 매니저 | 게임 상태 관리 (시작, 플레이, 일시정지, 게임오버) |
| 2 | 플레이어 시스템 | 캐릭터 이동, 체력, 피격, 사망 |
| 3 | 적(Enemy) 시스템 | 적 생성(스폰), AI 이동, 공격, 사망 |
| 4 | 무기 시스템 | 자동 공격, 다양한 무기 종류, 무기 레벨업 |
| 5 | 경험치/레벨업 시스템 | 보석 드롭, 흡수, 레벨업 UI, 선택지 제공 |
| 6 | 충돌(Collision) 시스템 | 투사체↔적, 적↔플레이어 충돌 판정 |

#### B. 메타 시스템 (Meta Systems) - 판 밖의 진행
| 순서 | 시스템 | 설명 |
|------|--------|------|
| 7 | 스테이지 시스템 | 시간 경과에 따른 난이도 증가, 보스 스폰 |
| 8 | 패시브 아이템 | 이동속도, 공격력 등 스탯 업그레이드 |
| 9 | 영구 업그레이드 | 골드로 구매하는 영구 강화 (메타 진행) |
| 10 | 캐릭터 선택 | 여러 캐릭터, 각각 고유 능력 |

#### C. 프레젠테이션 (Presentation) - 보여주기
| 순서 | 시스템 | 설명 |
|------|--------|------|
| 11 | UI/HUD | 체력바, 경험치바, 타이머, 킬 카운트 |
| 12 | 이펙트/파티클 | 공격 이펙트, 피격 이펙트, 레벨업 이펙트 |
| 13 | 사운드 | BGM, 효과음 (공격, 피격, 레벨업, 보석 획득) |
| 14 | 카메라 | 플레이어 추적 카메라, 화면 흔들림 |

#### D. 데이터 관리 (Data)
| 순서 | 시스템 | 설명 |
|------|--------|------|
| 15 | 데이터 테이블 | 무기/적/아이템 스탯을 외부 데이터로 관리 |
| 16 | 세이브/로드 | 영구 업그레이드, 해금 상태 저장 |

### 개발 순서 권장
```
1주차: 플레이어 이동 + 적 스폰 + 자동 공격 (프로토타입 확장)
2주차: 경험치/레벨업 + 무기 종류 추가 (3~4종)
3주차: 패시브 아이템 + 스테이지 타이머 + 보스
4주차: 캐릭터 선택 + 영구 업그레이드 + UI 완성
5주차: 이펙트, 사운드, 폴리싱
6주차: 테스트 + 버그 수정 + 배포 준비
```

## 4단계: 폴리싱 (Polishing)

> 게임을 "좋은 게임"에서 "완성된 게임"으로 만드는 단계

### 해야 할 일
- 버그 수정 (플레이테스트 반복)
- 밸런스 조정 (무기별 강약, 난이도 곡선)
- 화면 흔들림, 히트스탑 등 "게임 주스(Game Juice)" 추가
- UI/UX 개선 (직관적인 메뉴, 명확한 피드백)
- 성능 최적화 (몬스터 100마리 이상일 때 프레임 유지)

---

## 5단계: 출시/배포 (Release)

> 세상에 공개하는 단계

### 옵션
- **itch.io**: 무료 배포 가능, 인디 게임 플랫폼 (연습용으로 최적)
- **Steam**: 등록비 $100, 가장 큰 PC 게임 시장
- **웹 배포**: GitHub Pages 등으로 브라우저에서 바로 플레이

## Vampire Survivors 클론의 전체 구조도

```
VampireSurvivors-Clone/
├── docs/                        # 문서 (기획서, 용어집 등)
├── assets/                      # 리소스 파일 (향후 사용)
├── src/                         # 소스 코드
│   ├── main.js                  #   게임 시작점
│   ├── core/                    #   핵심 엔진
│   │   ├── Game.js              #     사령탑 (9종 상태 관리)
│   │   ├── Player.js            #     플레이어
│   │   ├── Input.js             #     키보드/마우스 입력
│   │   ├── Camera.js            #     카메라/흔들림
│   │   └── DebugMode.js         #     디버그 도구 (F1)
│   ├── entities/                #   게임 객체
│   │   ├── Enemy.js             #     적 (onDeath 통합)
│   │   ├── Gem.js               #     경험치 보석
│   │   ├── Projectile.js        #     투사체
│   │   ├── DamageText.js        #     데미지 숫자
│   │   └── Chest.js             #     보물상자
│   ├── weapons/                 #   무기 (기본 8 + 진화 8)
│   ├── systems/                 #   게임 시스템
│   │   ├── EnemySpawner.js      #     웨이브/보스 스폰
│   │   ├── CollisionSystem.js   #     충돌 판정
│   │   ├── ExpSystem.js         #     경험치/레벨업
│   │   ├── EvolutionSystem.js   #     무기 진화
│   │   └── UpgradeSystem.js     #     영구 업그레이드
│   ├── ui/                      #   화면 표시 (9개)
│   │   ├── HUD.js, MenuUI.js, CharSelectUI.js
│   │   ├── LevelUpUI.js, EvolutionUI.js
│   │   ├── PauseUI.js, UpgradeShopUI.js
│   │   └── GameOverUI.js, VictoryUI.js
│   ├── data/
│   │   └── config.js            #     모든 게임 수치 통합
│   └── utils/                   #   유틸리티
│       ├── MathUtils.js         #     수학 함수
│       ├── ObjectPool.js        #     오브젝트 풀링
│       ├── ErrorGuard.js        #     에러 처리
│       └── Storage.js           #     로컬 저장/로드
├── index.html                   # 게임 실행 진입점
└── README.md                    # 프로젝트 설명
```

## 추천 개발 도구

### 게임 엔진 (택 1)

| 엔진 | 난이도 | 언어 | 추천 이유 |
|------|--------|------|----------|
| **HTML5 Canvas + JavaScript** | ★★☆ | JavaScript | 브라우저에서 바로 실행, 설치 불필요, 웹 배포 쉬움 |
| **Phaser.js** | ★★☆ | JavaScript | 2D 게임 전용 웹 프레임워크, 기능이 풍부함 |
| **Godot** | ★★★ | GDScript | 무료 오픈소스 엔진, 2D 성능 우수, 경량 |
| **Unity** | ★★★★ | C# | 업계 표준, 에셋 스토어 풍부, 학습 자료 많음 |

### 초보자 추천: **Phaser.js** 또는 **순수 HTML5 Canvas**
- 이유: 별도 설치 없이 VS Code + 브라우저만으로 개발 가능
- Vampire Survivors 원작도 웹 기술(JavaScript) 기반으로 시작했음
- 결과물을 바로 웹에서 공유 가능

### 그 외 필요한 도구
| 용도 | 도구 | 설명 |
|------|------|------|
| 코드 편집 | VS Code | 이미 사용 중! |
| 도트 그래픽 | Aseprite, Piskel(무료) | 캐릭터, 몬스터 스프라이트 제작 |
| 사운드/음악 | BFXR(무료), Audacity | 효과음 생성, 음향 편집 |
| 버전 관리 | Git + GitHub | 코드 백업 및 이력 관리 |
| 무료 에셋 | OpenGameArt, itch.io | 남이 만든 그래픽/사운드 활용 |
