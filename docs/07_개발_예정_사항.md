## 도구 & 기술 도입 예정

### Claude Code 커스텀 에이전트 (agents)

> **현재**: 불필요 (Phase 1~2는 기본 도구로 충분)
> **도입 시점**: Phase 3 이후, 반복 작업이 많아질 때

| 에이전트 | 역할 | 도입 시점 | 상태 |
|----------|------|-----------|------|
| balance-tester | 밸런스 수치 자동 검증 (DPS 계산, 생존시간 시뮬레이션) | Phase 3 | PLANNED |
| sprite-validator | 스프라이트 규격 검사 (크기, 포맷, 네이밍) | Phase 3 | PLANNED |
| build-packager | 빌드 자동화 (웹/Electron/모바일) | Phase 4 | PLANNED |
| release-checker | 배포 전 체크리스트 자동 검증 | Phase 4 | PLANNED |
| perf-profiler | 프레임 드롭 구간 분석 & 리포트 | Phase 3 | PLANNED |

```
도입 기준:
- 같은 작업을 3번 이상 수동으로 반복했을 때
- 실수 가능성이 높은 반복 검증 작업이 생겼을 때
- "이거 자동화하면 좋겠다" 싶을 때

→ 그때 agents.md 파일을 만들고 에이전트 정의 시작
```

## 우선순위 가이드

```
P0 (필수)    → 이것 없이는 다음 Phase로 갈 수 없음
P1 (중요)    → 게임 품질에 직접 영향, 해당 Phase 내 완료 목표
P2 (하면좋음) → 있으면 좋지만, 일정 부족 시 다음으로 미룸
P3 (나중에)   → 아이디어 단계, 시간 여유 있을 때

원칙:
- P0부터 순서대로 처리
- 한 Phase의 P0가 모두 DONE이어야 다음 Phase로 진행
- DEFERRED 항목은 사유를 반드시 기록
```
