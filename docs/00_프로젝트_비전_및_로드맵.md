## 핵심 목표 (왜 이걸 하는가?)

### 1차 목표: 게임 개발의 전체 사이클을 체험한다
```
기획 → 개발 → 테스트 → 배포 → 운영 → 유지보수

"게임 하나를 처음부터 끝까지" 경험하는 것이 최우선 목표
완벽한 게임보다 완성된 게임이 100배 낫다
```

### 2차 목표: 실제 출시하여 수익화까지 경험한다
```
학습용으로 끝내지 않고, 실제 유저가 플레이하고 돈을 내는 경험까지 간다
금액의 크기가 중요한 게 아니라, "내 게임에 누군가 돈을 냈다"는 경험이 중요하다
```

### 3차 목표: 이 경험을 기반으로 오리지널 게임을 만든다
```
클론으로 배운 기술과 프로세스를 활용하여
차별화된 나만의 게임을 만들어 본격적으로 1인 개발자로 활동한다
```

## 개발 철학

### "부딪히면서 배운다"
- 이론 공부를 오래 하는 것보다, 일단 만들고 고치는 것이 빠르다
- 모르는 것은 만들면서 배우고, 틀리면 고치면 된다
- 실수는 학습의 재료다

### "작게 시작하고, 점점 키운다"
- 처음부터 대작을 만들지 않는다
- 핵심 재미부터 확인하고, 살을 붙여나간다
- 매 단계마다 "실행 가능한 상태"를 유지한다

### "실무처럼 한다"
- 혼자여도 문서를 쓰고, 규칙을 지키고, 체계적으로 개발한다
- 나중에 팀이 되거나 다른 프로젝트를 할 때 그대로 써먹을 수 있도록

## 전체 로드맵 (Phase별 계획)

### Phase 0: 기반 구축 ✅ 완료
```
[x] 프로젝트 구조 설계
[x] 규칙 문서 작성 (CLAUDE.md, ARCHITECTURE.md, CONVENTIONS.md)
[x] 개발 환경 세팅 (VS Code + 폴더 구조)
[x] 게임 개발 용어 학습 (용어집)
```

### Phase 1: 프로토타입 (핵심 재미 검증)
```
목표: "이동 + 자동공격 + 적 처치 + 레벨업"이 재미있는가?
기간 목표: 1~2주

[ ] 캐릭터가 WASD로 움직인다
[ ] 적이 화면 밖에서 생성되어 다가온다
[ ] 캐릭터가 자동으로 투사체를 발사한다
[ ] 적이 죽으면 보석을 떨어뜨린다
[ ] 보석을 모으면 레벨업한다
[ ] 레벨업 시 무기를 선택할 수 있다

완료 기준: 누군가에게 보여줬을 때 "한판만 더" 하고 싶게 만드는가?
```

### Phase 2: 본개발 (시스템 완성)
```
목표: 게임으로서 완결성을 갖춘다
기간 목표: 3~4주

[ ] 무기 종류 5개 이상
[ ] 패시브 아이템 시스템
[ ] 스테이지 타이머 (30분) + 난이도 곡선
[ ] 보스 몬스터
[ ] 캐릭터 선택 (3종 이상)
[ ] 영구 업그레이드 (골드 시스템)
[ ] 무기 진화 시스템
[ ] 사운드 & 이펙트

완료 기준: 처음부터 끝까지 한 판을 완주할 수 있는가?
```

### Phase 3: 폴리싱 & 차별화
```
목표: "클론"에서 "내 게임"으로 만든다
기간 목표: 2~3주

[ ] 오리지널 캐릭터/적 디자인 (저작권 문제 회피)
[ ] 독자적인 무기/스킬 아이디어 추가
[ ] 밸런스 조정 (플레이테스트 반복)
[ ] UI/UX 완성도 높이기
[ ] 성능 최적화 (적 200마리 이상에서 60fps)
[ ] 게임 타이틀, 로고, 아이콘 제작

완료 기준: Vampire Survivors를 모르는 사람이 봐도 "다른 게임"으로 느끼는가?
```

### Phase 4: 배포 & 출시
```
목표: 실제 유저가 플레이할 수 있게 만든다

[ ] 웹 버전 배포 (itch.io 또는 GitHub Pages) ← 1차 출시
[ ] 유저 피드백 수집 & 반영
[ ] Steam 출시 준비
    [ ] Steamworks 개발자 등록 ($100)
    [ ] 스토어 페이지 작성 (스크린샷, 트레일러, 설명)
    [ ] Steam 빌드 업로드 (Electron으로 .exe 패키징)
    [ ] Steam Next Fest 참가 (데모 출품)
    [ ] 위시리스트 확보
    [ ] 출시일 결정 & 출시
[ ] 모바일 출시 검토 (선택사항)
    [ ] PWA 또는 Capacitor로 모바일 앱 변환
    [ ] Google Play / App Store 등록

완료 기준: 실제 유저가 게임을 다운로드/구매하여 플레이했는가?
```

### Phase 5: 운영 & 유지보수
```
목표: 출시 후 게임을 살아있게 유지한다

[ ] 버그 리포트 대응 체계 구축
    - GitHub Issues 또는 Discord 서버
    - 버그 우선순위: 크래시 > 게임플레이 > 비주얼 > 기타
[ ] 패치 & 업데이트 사이클
    - 핫픽스: 크리티컬 버그 → 즉시 패치
    - 정기 업데이트: 2~4주 간격 콘텐츠 추가
    - 메이저 업데이트: 큰 기능 추가 (새 스테이지, 새 모드)
[ ] 커뮤니티 운영
    - Discord 서버 운영
    - 팬아트, 피드백 소통
    - 패치 노트 공개
[ ] 수익화 모델
    - B2P: 기본 가격 $2~3 (저가 정책)
    - DLC: 추가 캐릭터팩, 스테이지팩
    - 모바일: 광고 + 인앱 결제 (선택)
[ ] 데이터 분석 (선택)
    - 플레이 통계: 인기 무기, 평균 생존 시간, 이탈 구간
    - 유저 행동 분석으로 밸런스 조정

완료 기준: 출시 후 3개월간 꾸준한 업데이트가 이루어졌는가?
```

### Phase 6: 회고 & 다음 프로젝트

## 가상 플레이어 페르소나

> 실제 유저를 상정하고 그들의 입장에서 테스트한다

### 페르소나 1: "점심시간 게이머" 민수 (28세, 직장인)
```
플레이 패턴: 점심시간 15~20분, 출퇴근 지하철
좋아하는 것: 단순 조작, 빠른 성장 쾌감, "한판만 더"
싫어하는 것: 복잡한 튜토리얼, 긴 로딩, 어려운 조작
기대: 3천원으로 수십 시간 플레이할 수 있는 가성비
```

### 페르소나 2: "하드코어 수집가" 지연 (22세, 대학생)
```
플레이 패턴: 밤에 2~3시간 몰입 플레이
좋아하는 것: 100% 달성, 히든 요소 발견, 최적 빌드 연구
싫어하는 것: 콘텐츠 부족, 엔드게임 부재
기대: 모든 캐릭터 해금, 모든 무기 진화, 도전 과제 완료
```

### 페르소나 3: "스트리머" 현우 (25세, 유튜버)
```
플레이 패턴: 방송용으로 1~2시간 플레이
좋아하는 것: 시각적으로 화려한 장면, 예측 불가 전개, 시청자 반응
싫어하는 것: 반복적이고 단조로운 화면, 설명할 게 없는 게임
기대: "와 이거 뭐야!" 하는 순간이 자주 나오는 것
```
