# 게임 개발 용어집 (Game Development Glossary)

> 게임 개발 실무에서 자주 사용하는 용어를 카테고리별로 정리했습니다.
> 모르는 단어가 나오면 이 문서에서 찾아보세요!

---

## 1. 개발 프로세스 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| **기획서** | Game Design Document (GDD) | 게임의 모든 것을 적어놓은 설계 문서. 컨셉, 메카닉, UI 등 |
| **프로토타입** | Prototype | 핵심 기능만 최소한으로 만든 시험판. "이거 재밌나?" 검증용 |
| **MVP** | Minimum Viable Product | 최소 기능 제품. 가장 핵심적인 기능만 담은 버전 |
| **알파** | Alpha | 핵심 기능은 다 들어갔지만 버그가 많고 미완성인 상태 |
| **베타** | Beta | 거의 완성, 버그 수정과 밸런스 조정 단계 |
| **골드 마스터** | Gold Master (GM) | 출시 가능한 최종 버전 |
| **폴리싱** | Polishing | 게임을 매끄럽게 다듬는 작업 (이펙트, 사운드, 반응성 개선) |
| **빌드** | Build | 실행 가능한 형태로 만든 게임 파일 |
| **디버그/디버깅** | Debug/Debugging | 버그(오류)를 찾아서 고치는 작업 |
| **QA** | Quality Assurance | 품질 보증. 게임을 테스트하여 버그를 찾는 과정 |
| **얼리 액세스** | Early Access | 완성 전에 미리 판매하고, 유저 피드백을 받으며 개발 계속 |
| **포스트모템** | Post-mortem | 프로젝트 완료 후 "무엇이 잘됐고 무엇이 문제였나" 분석 |
| **이터레이션** | Iteration | 반복 개선. 만들고→테스트→수정→다시 만들기 사이클 |
| **마일스톤** | Milestone | 개발 중간 목표. "2주차까지 전투 시스템 완성" 같은 것 |
| **스코프** | Scope | 개발 범위. "스코프를 줄이자" = "기능을 줄이자" |
| **크런치** | Crunch | 출시 전 야근 폭탄 기간. 피해야 할 것 |

## 2. 게임 디자인 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| **게임 루프** | Game Loop | 한 판의 핵심 흐름. 플레이어가 반복하는 경험 사이클 |
| **코어 루프** | Core Loop | 가장 핵심적인 반복 구조. VS의 경우 "이동→전투→레벨업→반복" |
| **메타 게임** | Meta Game | 한 판 바깥의 진행 요소. 영구 업그레이드, 캐릭터 해금 등 |
| **메카닉** | Mechanic | 게임의 규칙/조작 방법. "자동 공격"이 하나의 메카닉 |
| **밸런스** | Balance | 게임의 균형. 무기별 강약, 난이도 등이 적절한가 |
| **난이도 곡선** | Difficulty Curve | 시간에 따라 게임이 어려워지는 정도의 그래프 |
| **파워 커브** | Power Curve | 플레이어가 강해지는 속도의 그래프 |
| **로그라이크** | Roguelike | 죽으면 처음부터, 랜덤 요소가 있는 장르 |
| **로그라이트** | Roguelite | 로그라이크 + 영구 업그레이드 (VS가 이 장르) |
| **뱀서라이크** | Survivors-like | Vampire Survivors 류의 자동공격 생존 게임 통칭 |
| **불릿헬** | Bullet Hell | 화면에 투사체가 가득한 장르 (탄막 슈팅) |
| **DPS** | Damage Per Second | 초당 피해량 |
| **AoE** | Area of Effect | 범위 공격. 일정 영역에 피해를 주는 것 |
| **버프/디버프** | Buff/Debuff | 능력치를 올려주는 것(버프) / 내려주는 것(디버프) |
| **스택** | Stack | 같은 효과가 중첩되는 것. "공격력 버프 3스택" |
| **쿨타임** | Cooldown (CD) | 스킬이나 공격을 다시 사용할 수 있을 때까지의 대기 시간 |
| **히트박스** | Hitbox | 충돌 판정 영역. 눈에 보이는 것과 다를 수 있음 |
| **넉백** | Knockback | 공격받았을 때 뒤로 밀려나는 것 |
| **무적 시간** | Invincibility Frame (i-frame) | 피격 후 잠시 무적이 되는 시간 |
| **게임 주스** | Game Juice/Feel | 화면흔들림, 히트스탑, 이펙트 등 "타격감"을 주는 연출 |
| **히트스탑** | Hitstop | 공격이 맞는 순간 잠깐 멈추는 연출. 타격감 증가 |
| **스폰** | Spawn | 적이나 아이템이 게임 세계에 생성되는 것 |
| **디스폰** | Despawn | 화면 밖이나 조건에 의해 사라지는 것 |
| **드롭** | Drop | 적이 죽을 때 아이템을 떨어뜨리는 것 |
| **어그로** | Aggro | 적이 플레이어를 인식하고 쫓아오는 것 |
| **웨이브** | Wave | 적이 한 무리씩 등장하는 단위 |

## 3. 프로그래밍 / 기술 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| **엔진** | Engine | 게임을 만드는 기본 프레임워크. Unity, Godot, Phaser 등 |
| **프레임워크** | Framework | 개발을 쉽게 해주는 코드 뼈대. 엔진보다 가벼운 개념 |
| **라이브러리** | Library | 특정 기능을 모아둔 코드 모음. 필요할 때 가져다 씀 |
| **IDE** | Integrated Development Environment | 코드 편집기 + 디버깅 도구. VS Code가 대표적 |
| **FPS** | Frames Per Second | 초당 화면 갱신 횟수. 60fps = 1초에 60번 그림 |
| **프레임** | Frame | 화면이 한 번 그려지는 단위. 60fps면 1프레임 = 약 16ms |
| **렌더링** | Rendering | 화면에 그래픽을 그리는 과정 |
| **업데이트 루프** | Update Loop | 매 프레임마다 실행되는 코드. 이동, 충돌, AI 등 처리 |
| **틱** | Tick | 게임 로직이 한 번 실행되는 단위 (= 1프레임의 로직) |
| **델타 타임** | Delta Time (dt) | 이전 프레임과 현재 프레임 사이의 시간 간격 |
| **스프라이트** | Sprite | 2D 게임에서 화면에 표시되는 이미지 한 장 |
| **스프라이트시트** | Spritesheet | 여러 애니메이션 프레임을 한 이미지에 모아놓은 것 |
| **타일맵** | Tilemap | 작은 이미지 조각(타일)을 반복해서 맵을 구성하는 방식 |
| **파티클** | Particle | 불꽃, 연기 등의 작은 효과 조각 |
| **콜리전/충돌** | Collision | 두 물체가 겹치는지 판단하는 것 |
| **AABB** | Axis-Aligned Bounding Box | 축에 정렬된 사각형 충돌 판정. 가장 기본적이고 빠른 방식 |
| **오브젝트 풀링** | Object Pooling | 객체를 미리 만들어두고 재활용하는 최적화 기법 |
| **가비지 컬렉션** | Garbage Collection (GC) | 안 쓰는 메모리를 자동 정리하는 것. 순간 끊김 원인이 되기도 |
| **싱글톤** | Singleton | 게임에서 하나만 존재해야 하는 관리자 패턴 (게임 매니저 등) |
| **컴포넌트** | Component | 기능 단위 조각. "이동 컴포넌트", "체력 컴포넌트" 등 |
| **엔티티** | Entity | 게임 세계의 모든 물체 통칭 (플레이어, 적, 아이템 등) |
| **씬** | Scene | 게임의 한 화면 단위. 메뉴 씬, 게임플레이 씬, 결과 씬 |
| **상태 머신** | State Machine | 상태에 따라 행동이 바뀌는 패턴. "대기→추적→공격→사망" |
| **벡터** | Vector | 방향과 크기를 가진 값. 이동, 속도 계산에 필수 |
| **정규화** | Normalize | 벡터의 방향은 유지하고 크기를 1로 만드는 것 |

## 4. 아트/그래픽 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| **픽셀아트** | Pixel Art | 점(픽셀) 단위로 그린 도트 그래픽 |
| **해상도** | Resolution | 화면 크기 (가로×세로 픽셀 수). 예: 1920×1080 |
| **에셋** | Asset | 게임에 사용되는 모든 리소스 (그래픽, 사운드, 데이터 등) |
| **텍스처** | Texture | 3D 모델이나 스프라이트에 입히는 이미지 |
| **애니메이션** | Animation | 여러 프레임을 연속 재생하여 움직이는 것처럼 보이게 함 |
| **프레임 애니메이션** | Frame Animation | 이미지를 바꿔가며 애니메이션하는 가장 기본 방식 |
| **트윈** | Tween | 시작값→끝값을 부드럽게 변화시키는 것 (이동, 크기, 투명도) |
| **이징** | Easing | 트윈의 속도 곡선. 천천히 시작→빨라지기, 바운스 등 |
| **알파** | Alpha (투명도) | 이미지의 투명도. 0=완전투명, 1=완전불투명 |
| **레이어** | Layer | 그래픽을 겹쳐 그리는 순서. 배경→몬스터→플레이어→UI |
| **카메라** | Camera | 게임 세계 중 화면에 보이는 영역을 결정 |
| **뷰포트** | Viewport | 실제 화면에 표시되는 영역 |
| **패럴랙스** | Parallax | 배경이 여러 층으로 다른 속도로 스크롤되는 입체 효과 |

## 5. 사운드 용어

### 기본 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| **BGM** | Background Music | 배경 음악 |
| **SFX** | Sound Effects | 효과음 (공격, 피격, UI 클릭 등) |
| **SE** | Sound Effect | SFX와 같은 의미 (일본식 표현) |
| **루프** | Loop | 음악이 반복 재생되는 것 |
| **페이드** | Fade | 소리가 점점 커지거나(Fade In) 작아지는(Fade Out) 것 |
| **프로시저럴 사운드** | Procedural Sound | 미리 녹음한 파일 없이 **코드로 소리를 실시간 생성**하는 방식. 용량 0, 무한 변형 가능 |
| **디바운스** | Debounce | 같은 소리가 너무 빠르게 반복될 때 일부를 무시하는 기법. 적 10마리 동시 타격 시 소리 10개가 겹치는 것 방지 |
| **마스터 볼륨** | Master Volume | 전체 소리의 최종 음량. 이 값 아래에 SFX/BGM 개별 볼륨이 있음 |
| **칩튠** | Chiptune | 8비트/16비트 게임기의 음원 칩으로 만든 복고풍 음악 |

### Web Audio API 용어

> 브라우저 내장 오디오 엔진. 외부 라이브러리 없이 사운드를 생성/재생할 수 있다.

| 용어 | 영문 | 설명 |
|------|------|------|
| **오디오 컨텍스트** | AudioContext | Web Audio API의 핵심 객체. 소리를 만들고 처리하는 **오디오 엔진** 역할. 게임에서 하나만 만들면 됨 |
| **오실레이터** | OscillatorNode | 소리 파형을 생성하는 장치. "삐~", "부~" 같은 기본 소리를 만듦. 프로시저럴 사운드의 핵심 |
| **게인 노드** | GainNode | 소리의 **볼륨을 조절**하는 노드. 0이면 무음, 1이면 원래 크기 |
| **오디오 버퍼** | AudioBuffer | 미리 녹음된 소리 파일(.mp3, .wav)을 메모리에 올려놓은 것. 파일 사운드 재생에 사용 |
| **데스티네이션** | Destination | 소리의 최종 출력지 = **스피커**. 모든 소리는 여기로 연결해야 들림 |
| **파형** | Waveform | 소리의 형태. sine(부드럽고 맑음), square(날카롭고 8비트), sawtooth(거칠고 강렬), triangle(중간) |
| **주파수** | Frequency (Hz) | 소리의 높낮이. 숫자가 높을수록 높은 음. 200Hz=낮은 펀치음, 1000Hz=높은 삐 소리 |
| **노드 연결** | Node Chain | 소리 노드를 체인처럼 연결하는 구조. `오실레이터 → 게인 → 필터 → 스피커` 순서로 연결 |
| **필터** | BiquadFilterNode | 소리에서 특정 주파수를 강조하거나 제거. 저역통과(lowpass)=둔탁한 소리, 고역통과(highpass)=날카로운 소리 |
| **램프** | Ramp (exponentialRampToValueAtTime) | 값을 서서히 변화시키는 것. 주파수를 점점 낮추면 "삐→부~" 효과, 볼륨을 0으로 내리면 페이드아웃 |

## 5-2. 시각 이펙트(VFX) & 화면 효과 용어

### 파티클 시스템

| 용어 | 영문 | 설명 |
|------|------|------|
| **파티클** | Particle | 작은 입자 하나. 수십~수백 개가 모여 폭발, 불꽃, 연기 같은 효과를 만듦 |
| **파티클 시스템** | Particle System | 파티클의 생성, 이동, 소멸을 관리하는 시스템. 이펙트의 핵심 엔진 |
| **파티클 풀** | Particle Pool | 파티클 객체를 미리 만들어두고 재활용하는 최적화 기법. 오브젝트 풀링의 파티클 버전 |
| **이미터** | Emitter | 파티클을 **방출(발사)**하는 지점. "적이 죽은 위치에서 파편을 발사"할 때의 그 위치 |
| **수명** | Lifetime | 파티클이 생성된 후 사라지기까지의 시간 (초). 보통 0.2~0.5초로 짧음 |
| **방사형 발사** | Radial Emission | 한 점에서 바깥으로 동그랗게 퍼져나가는 발사 패턴. 폭발 이펙트의 기본 |
| **중력** | Gravity | 파티클에 적용하는 아래 방향 힘. 0이면 무중력(직선 이동), 양수면 아래로 떨어짐 |

### 화면 효과 (Screen Effects)

| 용어 | 영문 | 설명 |
|------|------|------|
| **히트 프리즈** | Hit Freeze / Hitstop | 강한 타격 순간 게임을 **0.03~0.05초 멈추는** 효과. 타격의 "무게감"을 만든다. 업데이트만 멈추고 렌더는 계속 함 |
| **스크린 플래시** | Screen Flash | 화면 전체가 흰색(또는 다른 색)으로 **짧게 번쩍**이는 효과. 레벨업, 진화, 보스 사망 등 중요한 순간에 사용 |
| **카메라 쉐이크** | Camera Shake | 화면이 짧게 **흔들리는** 효과. 폭발, 보스 등장, 강한 타격 시 사용. 우리 게임에 이미 구현되어 있음 |
| **슬로우 모션** | Slow Motion / Bullet Time | 게임 속도를 일시적으로 **느리게** 만드는 효과 (예: 50% 속도 0.3초). 보스 처치의 결정적 순간에 사용 |
| **비네트** | Vignette | 화면 **가장자리를 어둡게** 만드는 효과. 체력이 낮을 때 위험 느낌, 또는 집중 유도에 사용 |
| **머즐 플래시** | Muzzle Flash | 무기 발사 시 **발사 지점에서 나오는 섬광**. 투사체가 발사된다는 시각적 피드백 |
| **글로우** | Glow | 물체 주변에 **빛나는 효과**를 주는 것. Canvas에서는 `shadowBlur`와 `shadowColor`로 구현 |
| **페이드** | Fade | 투명도(alpha)를 서서히 변화시키는 것. 파티클이 사라질 때 점점 투명해지는 게 대표적 |
| **오버레이** | Overlay | 게임 화면 **위에 반투명 레이어**를 깔아 효과를 주는 것. 일시정지 어둡게, 피격 시 빨간 틴트 등 |

### Game Feel (게임 느낌)

| 용어 | 영문 | 설명 |
|------|------|------|
| **게임 필** | Game Feel | 조작감 + 타격감 + 시각/청각 피드백이 어우러진 **"게임다운 느낌"**. 같은 기능이어도 Game Feel에 따라 재미가 달라짐 |
| **주스** | Juice | Game Feel을 높이는 **작은 연출들의 총칭**. 화면 흔들림, 파티클, 히트 프리즈, 사운드 등 |
| **피드백** | Feedback | 플레이어의 행동에 대한 **반응**. 적을 때리면 흔들림+소리+파편 = 좋은 피드백 |
| **임팩트** | Impact | 타격이나 폭발의 **무게감**. 히트 프리즈 + 카메라 쉐이크 + 파티클 + 사운드 = 임팩트 |
| **폴리싱** | Polishing | 기능은 완성된 상태에서 **이펙트/사운드/반응성**을 다듬어 퀄리티를 높이는 작업 |
| **스킨** | Skin | 기능은 같지만 **외형(색상, 모양, 이펙트)**만 다른 버전 |

## 7. Vampire Survivors 관련 특수 용어

| 용어 | 설명 |
|------|------|
| **자동 공격** | 플레이어 조작 없이 무기가 알아서 발사됨 |
| **진화(Evolution)** | 특정 무기 + 특정 패시브 조합으로 무기가 상위 버전으로 변하는 시스템 |
| **보석(Gem)** | 적이 죽으면 떨어지는 경험치 아이템 |
| **자석(Magnet)** | 보석을 끌어당기는 효과 |
| **사신(Death/Reaper)** | 30분 경과 시 등장하는 즉사 적. 사실상 게임 종료 트리거 |
| **아르카나(Arcana)** | 게임 시작 전 선택하는 특수 룰 변경 카드 |
| **금화(Gold Coin)** | 스테이지에서 획득하여 영구 업그레이드에 사용하는 재화 |
| **보물상자(Treasure Chest)** | 보스 처치 시 드롭, 무기 진화에 사용 |
