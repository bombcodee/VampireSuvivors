# 버그 트래커 (Bug Tracker)

> 발견된 버그를 추적하고 관리하는 문서입니다.
> 실무에서는 Jira, GitHub Issues 등의 도구를 쓰지만,
> 1인 개발에서는 이 문서로 충분합니다.
>
> 상태: OPEN(미해결) / FIXED(수정완료) / WONTFIX(수정 안 함) / DUPLICATE(중복)
> 심각도: Critical(크래시) / Major(게임플레이 영향) / Minor(사소함) / Cosmetic(외관)

---

## 버그 ID 규칙
- `BUG-XXX` 형식으로 순번 부여
- 한 번 부여된 번호는 재사용하지 않음

## FIXED (수정 완료)

### BUG-001: Garlic 무기 적 처치 시 보석 미드롭
- **심각도**: Major
- **발견일**: 2026-02-27
- **수정일**: 2026-02-27
- **증상**: Garlic으로 적을 죽여도 경험치 보석이 떨어지지 않고, 킬 카운트도 증가하지 않음
- **원인**: Garlic._areaAttack()에서 enemy.takeDamage() 반환값(사망 여부)을 무시하고 있었음. 사망한 적의 보석 드롭과 킬 처리 로직이 없었음
- **수정**: Garlic에 game 참조를 전달하여, 적 사망 시 보석 드롭 + 킬카운트 증가 + enemy.active = false 처리 추가
- **영향 파일**: `Garlic.js`, `Player.js`, `Game.js`

### BUG-002: 넉백 중 적이 동시에 전진
- **심각도**: Minor
- **발견일**: 2026-02-27
- **수정일**: 2026-02-27
- **증상**: 적이 넉백되면서 동시에 플레이어 방향으로 이동하여, 넉백 효과가 거의 보이지 않음
- **원인**: Enemy.update()에서 넉백 이동과 추적 이동이 동시에 적용됨
- **수정**: 넉백 중일 때는 추적 이동을 스킵하도록 조건 분기
- **영향 파일**: `Enemy.js`

### BUG-003: 넉백 감쇠 프레임레이트 의존
- **심각도**: Minor
- **발견일**: 2026-02-27
- **수정일**: 2026-02-27
- **증상**: 30fps에서 넉백이 60fps보다 오래 지속됨
- **원인**: `knockbackX *= 0.9`로 매 프레임 고정 감쇠 → fps에 따라 결과 다름
- **수정**: `Math.pow(0.005, dt)`로 프레임레이트 독립적 감쇠 적용
- **영향 파일**: `Enemy.js`

### BUG-004: 매 프레임 bind() 호출
- **심각도**: Cosmetic (성능)
- **발견일**: 2026-02-27
- **수정일**: 2026-02-27
- **증상**: 프레임마다 새로운 함수 객체 생성 → 불필요한 GC 부하
- **원인**: `requestAnimationFrame(this._gameLoop.bind(this))` 매 프레임 호출
- **수정**: constructor에서 한 번만 bind하고 재사용
- **영향 파일**: `Game.js`

### BUG-005: 미사용 import

- **발견일**: 2026-02-27
- **수정일**: 2026-02-27

- **영향 파일**: `Player.js`, `Enemy.js`, `HUD.js`, `LevelUpUI.js`

### BUG-006: BGM 동시 재생 (T스킵 + X즉사 시 구 BGM 안꺼짐)
- **심각도**: Major
- **발견일**: 2026-03-19
- **수정일**: 2026-03-19
- **증상**: Shift+T로 29:30 돌파 후 바로 30:00 돌파 또는 X즉사 시, 게임 BGM과 보스/게임오버 BGM이 동시에 재생됨
- **원인**: `stopBGM(true, 28)` 호출 시 `_bgmCurrentSource = null` 처리하지만 실제 audio 노드(`srcToStop`)는 28초 후 setTimeout에서 정지 예정. 이후 `playBGM()` 호출 시 `_bgmCurrentSource === null`로 stop 로직 스킵 → 구 BGM + 신 BGM 동시 재생
- **수정**: `_bgmFadingSource` 추적 변수 추가. `playBGM()` 시작 시 `_bgmFadingSource`가 있으면 즉시 정지
- **영향 파일**: `src/core/SoundManager.js`

### BUG-007: 디버그 L키 — HUD 레벨 미반영
- **심각도**: Minor
- **발견일**: 2026-03-19
- **수정일**: 2026-03-19
- **증상**: 디버그 L키 누르면 레벨업 선택 UI는 뜨지만, HUD 레벨 숫자가 올라가지 않음
- **원인**: `game.onLevelUp()` 만 호출하고 `player.level++` 없음. 정상 흐름은 `CollisionSystem`에서 `player.level++` 후 `onLevelUp()` 호출하는 구조
- **수정**: DebugMode.js L키 핸들러에 `player.level++` + `player.exp = 0` + `expToNext` 갱신 추가
- **영향 파일**: `src/core/DebugMode.js`
