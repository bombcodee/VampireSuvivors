# 기술 스택 & 도구 선택 가이드

## 우리가 선택할 기술 스택

### 게임 엔진: 순수 HTML5 Canvas + JavaScript

**왜 이걸 선택하는가?**
1. **설치할 것이 없다** - VS Code + 브라우저만 있으면 바로 시작
2. **원작과 같은 방식** - Vampire Survivors 원작도 JavaScript 기반
3. **결과를 바로 볼 수 있다** - 파일 저장 → 브라우저 새로고침 → 바로 확인
4. **웹으로 바로 공유 가능** - GitHub Pages로 무료 배포
5. **게임 개발의 원리를 배울 수 있다** - 엔진이 해주는 것을 직접 만들어보면 이해도가 높아짐

### 전체 기술 스택

```
┌─────────────────────────────────────────┐
│                 브라우저                  │
│  ┌─────────────────────────────────────┐│
│  │           HTML5 Canvas              ││  ← 그래픽 렌더링
│  │  ┌───────────────────────────────┐  ││
│  │  │     JavaScript (ES6+)         │  ││  ← 게임 로직
│  │  │  ┌─────────┐ ┌─────────────┐  │  ││
│  │  │  │ 게임엔진  │ │  에셋 관리   │  │  ││
│  │  │  │ (직접제작)│ │ (이미지,소리)│  │  ││
│  │  │  └─────────┘ └─────────────┘  │  ││
│  │  └───────────────────────────────┘  ││
│  └─────────────────────────────────────┘│
│  HTML 파일 (index.html)                  │
└─────────────────────────────────────────┘
```

## 개발 환경 세팅 순서

### Step 1: VS Code 확장 프로그램 설치
```
필수:
- Live Server          : 로컬 서버 실행 (자동 새로고침)
- ESLint               : JavaScript 문법 오류 검사

추천:
- Bracket Pair Colorizer : 괄호 짝 색상 구분
- GitLens              : Git 히스토리 쉽게 보기
```

### Step 2: 프로젝트 폴더 구조 생성
```
VampireSurvivors-Clone/
├── index.html          ← 게임 실행 파일 (브라우저에서 이걸 열면 게임 시작)
├── style.css           ← 화면 스타일
├── src/                ← 소스 코드 폴더
│   ├── main.js         ← 게임 시작점
│   ├── core/           ← 핵심 시스템
│   ├── entities/       ← 게임 객체 (플레이어, 적, 무기)
│   ├── systems/        ← 게임 시스템 (레벨업, 스폰 등)
│   └── ui/             ← 화면 표시
├── assets/             ← 리소스
│   ├── sprites/        ← 이미지
│   ├── audio/          ← 소리
│   └── data/           ← 데이터 (JSON)
└── docs/               ← 문서 (지금 이 파일들)
```

### Step 3: 개발 시작!
1. `index.html` 생성 → Canvas 태그 삽입
2. `main.js` 생성 → 게임 루프 작성
3. 브라우저에서 Live Server로 실행
4. 코드 수정 → 저장 → 자동 새로고침으로 결과 확인

## 핵심 기술 개념 미리보기

### HTML5 Canvas란?
- 브라우저에서 그림을 그릴 수 있는 도화지
- JavaScript로 사각형, 원, 이미지 등을 그릴 수 있음
- 게임의 화면 출력을 담당

```javascript
// 예시: 캔버스에 빨간 사각형 그리기
const canvas = document.getElementById('gameCanvas');
const ctx = canvas.getContext('2d');
ctx.fillStyle = 'red';
ctx.fillRect(100, 100, 50, 50);  // x, y, 가로, 세로
```

### 게임 루프란?
- 게임이 실행되는 동안 계속 반복되는 핵심 구조
- 매 프레임(1/60초)마다: 입력 확인 → 상태 업데이트 → 화면 그리기

```javascript
// 게임 루프 기본 구조
function gameLoop() {
    handleInput();   // 1. 키보드/마우스 입력 확인
    update();        // 2. 게임 상태 업데이트 (이동, 충돌 등)
    render();        // 3. 화면에 그리기
    requestAnimationFrame(gameLoop);  // 다음 프레임 예약
}
```

## 배포: GitHub Pages

### 왜 GitHub Pages인가?

| 항목 | 내용 |
|------|------|
| **비용** | 무료 (Public 레포지토리 기준) |
| **방식** | GitHub에 push하면 1~2분 후 자동 반영 |
| **주소** | `https://[유저명].github.io/[저장소명]/` |
| **조건** | 정적 파일(HTML/CSS/JS)만 지원 — 우리 게임과 딱 맞음 |

### 배포 설정 방법

1. GitHub 레포지토리 → **Settings** 탭
2. 왼쪽 메뉴 → **Pages**
3. Source: `Deploy from a branch`
4. Branch: `master` / Folder: `/ (root)`
5. **Save** 클릭 → 1~2분 대기

### 현재 배포 정보

- **저장소**: https://github.com/bombcodee/VampireSuvivors
- **게임 링크**: https://bombcodee.github.io/VampireSuvivors/
- **배포 시작일**: 2026-03-17

### 업데이트 흐름

```
코드 수정
   ↓
git commit -m "feat: ..."
   ↓
git push origin master
   ↓
GitHub Pages 자동 빌드 (1~2분)
   ↓
링크에서 최신 버전 확인
```

## 나중에 확장할 때

처음에는 순수 Canvas로 시작하고, 나중에 필요하면:
- **Phaser.js**: 2D 게임 전문 프레임워크로 이전 가능
- **Electron**: 웹 게임을 데스크톱 앱(.exe)으로 변환
- **Capacitor**: 웹 게임을 모바일 앱으로 변환
- **Steam**: Electron으로 포장 후 Steam에 출시 가능
