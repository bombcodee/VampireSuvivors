# 사운드 & 이펙트 가이드 (Sound & VFX Guide)

> 이 프로젝트의 사운드와 시각 효과 설계 방향, 기술 구현 방법, 구조를 정리한 문서입니다.
> 게임 개발에서 "게임 느낌(Game Feel)"을 만드는 핵심 요소입니다.

---

## 1. 디자인 방향: Sleek + Trendy 하이브리드

### 왜 이 방향인가?

| 검토한 스타일 | 채택 여부 | 이유 |
|---|---|---|
| 도트/픽셀 | ✗ | 현재 Canvas 도형 기반 → 스프라이트 전환 비용 높음 |
| 세련된(Sleek) | **◎ 베이스** | 현재 코드와 자연스럽게 연결, 리소스 제로 |
| 트렌디(Trendy) | **◎ 포인트** | 글로우/파티클/쉐이크로 임팩트 강화 |
| 고품질(Hi-Fi) | ✗ | WebGL 전환 + 에셋 구매 필요, 비용 과다 |

**결론**: 깔끔한 기하학적 도형을 기본으로, 핵심 순간(타격, 처치, 레벨업, 진화)에만 글로우/파티클/화면 효과를 얹는다.

### 핵심 원칙

```
1. 코드만으로 구현 (외부 에셋 최소화)
2. 성능 우선 (파티클 풀링, 오브젝트 재사용)
3. 과하지 않게 (시인성 > 화려함)
4. 점진적 개선 (기본 → 파티클 → 화면 효과 순)
```

## 2. 사운드 시스템

### 2-1. 기술: Web Audio API

브라우저 내장 오디오 엔진. 외부 라이브러리 없이 사용 가능.

**핵심 개념:**

```
AudioContext (오디오 엔진)
  ├→ OscillatorNode (소리 생성기: 삐~, 부~, 찌직~)
  ├→ GainNode (볼륨 조절)
  ├→ BiquadFilterNode (소리 필터: 저음 강조, 고음 제거 등)
  └→ destination (스피커 출력)
```

- **OscillatorNode**: 파형(sine, square, sawtooth, triangle)을 생성. 게임 효과음의 기본
- **GainNode**: 소리 크기 조절. 페이드인/아웃에 사용
- **AudioBuffer**: 미리 녹음된 소리 파일(.mp3, .wav)을 재생

### 2-2. 프로시저럴 사운드 vs 파일 사운드

| 방식 | 장점 | 단점 | 우리 게임에서 |
|---|---|---|---|
| **프로시저럴** (코드로 생성) | 파일 없음, 용량 0, 무한 변형 | 자연스러운 소리 어려움 | 히트음, UI음, 간단한 효과 |
| **파일 재생** (.mp3/.wav) | 고품질, 자연스러움 | 파일 용량, 라이선스 | BGM, 복잡한 효과음 |

**우리 전략**: 효과음은 프로시저럴 우선, BGM만 파일 사용 (나중에)

### 2-3. SoundManager 설계

```
src/core/SoundManager.js
  ├→ _ctx: AudioContext        ← 오디오 엔진 (싱글톤)
  ├→ _masterGain: GainNode     ← 마스터 볼륨
  ├→ _sfxGain: GainNode        ← 효과음 볼륨
  ├→ _bgmGain: GainNode        ← BGM 볼륨
  │
  ├→ play(soundId)             ← 효과음 재생 (ID 기반)
  ├→ setVolume(type, value)    ← 볼륨 조절 (0.0~1.0)
  ├→ mute() / unmute()         ← 음소거 토글
  │
  └→ 내장 사운드 정의:
      ├→ 'hit'       : 짧은 펀치음 (square wave, 0.05초)
      ├→ 'kill'      : 적 처치음 (noise burst, 0.1초)
      ├→ 'levelup'   : 레벨업 종소리 (sine 화음, 0.3초)
      ├→ 'evolve'    : 진화 팡파레 (ascending chord, 0.5초)
      ├→ 'pickup'    : 보석 수집음 (high ping, 0.05초)
      ├→ 'chest'     : 상자 오픈 (metallic, 0.2초)
      ├→ 'bosswarn'  : 보스 경고음 (low rumble, 1.0초)
      └→ 'ui_click'  : UI 클릭음 (soft click, 0.03초)
```

### 2-4. 프로시저럴 사운드 만드는 법 (예시)

```javascript
// 히트음 예시: 짧은 square wave + 빠른 페이드아웃
_playHit() {
    const osc = this._ctx.createOscillator();
    const gain = this._ctx.createGain();

    osc.type = 'square';                    // 파형: 딱딱한 사각파
    osc.frequency.value = 200;              // 주파수: 200Hz (낮은 펀치음)
    osc.frequency.exponentialRampToValueAtTime(50, now + 0.05);  // 빠르게 낮아짐

    gain.gain.value = 0.3;                  // 볼륨
    gain.gain.exponentialRampToValueAtTime(0.001, now + 0.05);   // 빠른 페이드아웃

    osc.connect(gain).connect(this._sfxGain);
    osc.start();
    osc.stop(now + 0.05);                   // 0.05초 후 정지
}
```

**파형별 느낌:**
| 파형 | 소리 느낌 | 용도 |
|---|---|---|
| `sine` | 부드럽고 맑음 (♪~) | 보석 수집, 레벨업, UI |
| `square` | 날카롭고 8비트 (삐~) | 히트음, 대시 |
| `sawtooth` | 거칠고 강렬 (지지직~) | 번개, 보스 경고 |
| `triangle` | sine과 square 중간 | 범용 |

### 2-5. 사운드 재생 타이밍 (게임 코드에서)

```javascript
// CollisionSystem.js — 적 피격 시
if (isDead) {
    game.sound.play('kill');    // 처치음
} else {
    game.sound.play('hit');     // 히트음
}

// ExpSystem.js — 레벨업 시
game.sound.play('levelup');

// EvolutionSystem.js — 진화 시
game.sound.play('evolve');
```

> **주의**: 동시에 너무 많은 소리가 나면 귀가 아프다.
> 같은 소리는 0.05초 내 중복 재생을 방지하는 **디바운스** 필요.

## 3. 시각 이펙트(VFX) 시스템

### 3-1. 파티클 시스템

게임에서 "폭발", "불꽃", "먼지" 같은 효과를 만드는 핵심 시스템.
수십~수백 개의 작은 입자(파티클)가 흩어지면서 시각 효과를 만든다.

**파티클 하나의 속성:**

```javascript
{
    x, y,           // 위치
    vx, vy,         // 속도 (방향 + 세기)
    life,           // 남은 수명 (초)
    maxLife,         // 초기 수명 (페이드 계산용)
    size,            // 크기
    color,           // 색상
    gravity,         // 중력 영향 (0이면 무중력)
}
```

**파티클 풀 (ParticlePool):**

```
src/effects/ParticlePool.js
  ├→ _particles: Array[200]    ← 미리 생성된 파티클 객체
  ├→ emit(config)              ← 파티클 방출 (위치, 수량, 색상, 속도 등)
  ├→ update(dt)                ← 모든 파티클 위치/수명 갱신
  └→ render(ctx, camera)       ← 활성 파티클만 렌더링
```

### 3-2. 이펙트 종류별 설계

#### 적 처치 파편 (Death Burst)
```
- 발생: 적 사망 시
- 파티클 수: 4~6개
- 색상: 적의 COLOR
- 속도: 바깥으로 방사형 (100~200px/s)
- 수명: 0.3~0.5초
- 중력: 약간 (아래로 떨어지는 느낌)
- 크기: 2~4px, 시간에 따라 축소
```

#### 보석 수집 반짝임 (Gem Sparkle)
```
- 발생: 보석 흡수 시
- 파티클 수: 3개
- 색상: 보석 COLOR (#69f0ae)
- 속도: 위쪽으로 느리게 (30~60px/s)
- 수명: 0.2~0.3초
- 크기: 1~2px
```

#### 레벨업 원형 확산 (Level Up Ring)
```
- 발생: 레벨업 시
- 방식: 파티클 아닌 원형 렌더링
- 동작: 반지름 0 → 200px로 확대, 알파 1.0 → 0.0 페이드
- 색상: 흰색 + 금색 테두리
- 지속: 0.4초
```

#### 진화 연출 (Evolution Flash)
```
- 발생: 무기 진화 선택 시
- 동작: 화면 전체 흰색 플래시 (0.1초) + 파티클 방출
- 파티클: 금색, 플레이어 위치에서 12개 방사
- 추가: 화면 흔들림 (Camera shake)
```

#### 무기 머즐 플래시 (Muzzle Flash)
```
- 발생: 투사체 무기 발사 시
- 방식: 발사 위치에 작은 원형 플래시 (0.05초)
- 색상: 무기의 COLOR
- 크기: 투사체 size의 2배 → 0으로 축소
```

### 3-3. 화면 효과 (Screen Effects)

이미 Camera.js에 shake가 있으므로, 이를 확장한다.

| 효과 | 설명 | 트리거 |
|---|---|---|
| **Camera Shake** | 화면 흔들림 (이미 구현됨) | 보스 등장, 강한 타격 |
| **Hit Freeze** | 0.03~0.05초 게임 정지 | 강력한 일격 (진화 무기 등) |
| **Screen Flash** | 화면 전체 흰색/색상 깜빡임 | 레벨업, 진화, 보스 사망 |
| **Slow Motion** | 0.3초간 게임 속도 50% | 보스 처치 마지막 일격 |
| **Vignette** | 화면 가장자리 어두워짐 | 체력 낮을 때 위험 표시 |

**Hit Freeze(히트 프리즈)란?**
게임을 아주 짧은 시간(0.03초) 멈추는 효과. 타격의 "무게감"을 만든다.
원작 뱀서에도 있고, Hades, Hollow Knight 등 액션 게임의 필수 기법.

```javascript
// Game.js에서 구현
hitFreeze(duration = 0.03) {
    this._freezeTimer = duration;
}

// 메인 루프에서
update(dt) {
    if (this._freezeTimer > 0) {
        this._freezeTimer -= dt;
        return;  // 업데이트 스킵 (렌더만 함)
    }
    // ... 정상 업데이트
}
```

## 4. 파일 구조 계획

```
src/
  core/
    SoundManager.js     ← 사운드 엔진 (프로시저럴 + 파일 재생)
  effects/
    ParticlePool.js     ← 파티클 풀 (오브젝트 풀 패턴)
    ScreenEffects.js    ← 화면 효과 (flash, vignette)
  data/
    config.js           ← +SOUNDS (볼륨, 디바운스), +PARTICLES (각 이펙트 설정)
```

### config.js 추가 예정 섹션

```javascript
// ===== 사운드 설정 =====
export const SOUNDS = {
    MASTER_VOLUME: 0.5,
    SFX_VOLUME: 0.7,
    BGM_VOLUME: 0.3,
    DEBOUNCE_TIME: 0.05,     // 같은 소리 최소 간격 (초)
};

// ===== 파티클 설정 =====
export const PARTICLES = {
    MAX_COUNT: 200,           // 최대 동시 파티클 수
    DEATH_BURST: { count: 5, speed: 150, life: 0.4, gravity: 80 },
    GEM_SPARKLE: { count: 3, speed: 50, life: 0.25, gravity: 0 },
    LEVELUP_RING: { duration: 0.4, maxRadius: 200 },
    EVOLUTION_FLASH: { count: 12, speed: 200, life: 0.5, gravity: 0 },
};
```

## 5. 구현 순서

```
Step 1: SoundManager (프로시저럴 효과음 8종)
  → 히트, 처치, 보석, 레벨업, 진화, 상자, 보스경고, UI클릭
  → Game.js에 연결, 각 시스템에서 play() 호출

Step 2: ParticlePool (기본 파티클 시스템)
  → 적 처치 파편, 보석 반짝임
  → Game.js에 연결, CollisionSystem에서 emit() 호출

Step 3: ScreenEffects (화면 효과)
  → Hit Freeze, Screen Flash
  → 레벨업/진화/보스 처치 시 적용

Step 4: 고급 이펙트 (선택)
  → Slow Motion, Vignette
  → 무기별 특화 이펙트
```

## 6. 용어 정리

| 용어 | 설명 |
|---|---|
| **프로시저럴 (Procedural)** | 미리 만들어둔 파일 대신 코드로 실시간 생성하는 방식 |
| **파티클 (Particle)** | 작은 입자. 수십 개가 모여 폭발, 불꽃 등의 이펙트를 만듦 |
| **오실레이터 (Oscillator)** | 소리 파형을 생성하는 장치 (Web Audio API의 핵심) |
| **게인 (Gain)** | 소리 크기(볼륨) 조절 |
| **디바운스 (Debounce)** | 같은 이벤트가 너무 빠르게 반복될 때 일부를 무시하는 기법 |
| **히트 프리즈 (Hit Freeze)** | 타격 순간 게임을 아주 짧게 멈춰 무게감을 주는 기법 |
| **비네트 (Vignette)** | 화면 가장자리를 어둡게 하는 효과 (위험/긴장감 연출) |
| **머즐 플래시 (Muzzle Flash)** | 무기 발사 시 총구에서 나오는 섬광 효과 |
| **Game Feel** | 조작감, 타격감, 시각/청각 피드백이 어우러진 "게임다운 느낌" |
