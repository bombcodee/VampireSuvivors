# ARCHITECTURE.md - 게임 아키텍처 설계서

> 이 문서는 게임의 전체 구조와 각 시스템이 어떻게 연결되는지를 정의합니다.

## 전체 아키텍처 다이어그램

```
                        ┌─────────────┐
                        │  index.html │  ← 진입점 (Canvas 생성)
                        └──────┬──────┘
                               │
                        ┌──────▼──────┐
                        │   main.js   │  ← 게임 초기화 & 시작
                        └──────┬──────┘
                               │
                 ┌─────────────▼─────────────┐
                 │        Game (싱글톤)        │  ← 게임 전체를 관리하는 중앙 매니저
                 │  - 상태: 9종 (MENU~VICTORY)│
                 │  - 모든 시스템을 소유       │
                 └─────────────┬─────────────┘
                               │ 매 프레임마다
                    ┌──────────┼──────────┐
                    ▼          ▼          ▼
              ┌─────────┐ ┌────────┐ ┌────────┐
              │  Input   │ │ Update │ │ Render │
              │ (입력)   │ │ (로직) │ │ (그리기)│
              └─────────┘ └───┬────┘ └────────┘
                              │
          ┌───────────┬───────┼───────┬────────────┐
          ▼           ▼       ▼       ▼            ▼
    ┌──────────┐ ┌────────┐ ┌─────┐ ┌──────┐ ┌─────────┐
    │  Player  │ │ Enemy  │ │Weapon│ │ Exp  │ │  Stage  │
    │  System  │ │ System │ │System│ │System│ │  System │
    └──────────┘ └────────┘ └─────┘ └──────┘ └─────────┘
```

## 게임 상태 흐름 (State Machine)

```
┌────────┐    시작     ┌────────┐   레벨업    ┌─────────┐
│  MENU  │ ─────────→ │  PLAY  │ ─────────→ │ LEVELUP │
│ (메뉴) │            │(플레이)│ ←───────── │(선택 UI)│
└────────┘            └───┬────┘   선택완료   └─────────┘
                          │
                     ESC키 │ 사망
              ┌────────┐  │  ┌──────────┐
              │ PAUSE  │←─┼─→│ GAMEOVER │
              │(일시정지)│  │  │(게임오버) │
              └────────┘  │  └─────┬────┘
                          │        │ 재시작
                          │        ▼
                          └──── MENU로
```

### 상태별 동작

| 상태 | Update | Render | Input |
|------|--------|--------|-------|
| MENU | X | 메뉴 화면 | Enter=시작, C=캐릭터선택, S=상점 |
| CHARSELECT | X | 캐릭터 선택 화면 | ←→=선택, Enter=확정 |
| SHOP | X | 업그레이드 상점 | ↑↓=선택, Enter=구매, ESC=닫기 |
| PLAY | 모든 시스템 | 게임 화면 | 이동 (WASD/방향키) |
| LEVELUP | X (멈춤) | 게임 화면 + 선택 UI | 마우스 클릭 (무기 선택) |
| EVOLUTION | X (멈춤) | 게임 화면 + 진화 UI | 클릭 (진화 선택) |
| PAUSE | X (멈춤) | 게임 화면 + 일시정지 표시 | ESC (재개) |
| GAMEOVER | X | 결과 화면 | Enter=메뉴 |
| VICTORY | X | 클리어 화면 | Enter=메뉴 |

## 핵심 시스템 상세

### 1. Game (중앙 매니저)
```
역할: 모든 것을 관리하는 "사령탑"
- 게임 상태 관리 (MENU, PLAY, PAUSE 등)
- 모든 시스템 초기화 및 소유
- 게임 루프 실행 (매 프레임 update → render 호출)
- 전역 데이터 접근 창구 (시간, 점수, 킬수 등)

의존: Canvas, Input, 모든 System
```

### 2. Player (플레이어)
```
역할: 플레이어 캐릭터 관리
- 위치(x, y), 이동 속도
- 체력(HP), 피격 처리, 무적 시간
- 경험치, 레벨
- 보유 무기 목록
- 보유 패시브 아이템 목록

의존: Input (이동 명령), WeaponSystem (무기 장착)
```

### 3. EnemySystem (적 시스템)
```
역할: 적의 생성, 이동, 관리
- 적 스폰 (화면 밖에서 생성)
- AI 이동 (플레이어를 향해 직선 이동)
- 피격 처리 (HP 감소, 사망)
- 사망 시 경험치(보석) 드롭
- 오브젝트 풀링으로 성능 관리

의존: Player (추적 대상), ExpSystem (보석 드롭)
```

### 4. WeaponSystem (무기 시스템)
```
역할: 무기 발사, 투사체 관리
- 무기별 자동 공격 (쿨타임 기반)
- 투사체 생성 및 이동
- 투사체-적 충돌 판정
- 무기 레벨업 (데미지, 발사 수, 범위 증가)
- 오브젝트 풀링으로 투사체 관리

의존: Player (발사 위치), EnemySystem (충돌 대상)
```

### 5. ExpSystem (경험치 시스템)
```
역할: 경험치 보석 관리 및 레벨업
- 보석 오브젝트 생성 및 관리
- 보석 흡수 판정 (플레이어 근접 시)
- 경험치 누적 및 레벨업 판정
- 레벨업 시 게임 상태를 LEVELUP으로 전환
- 레벨업 선택지 생성 (무기/패시브 랜덤 3개)

의존: Player (경험치 흡수), Game (상태 전환)
```

### 6. StageSystem (스테이지 시스템)
```
역할: 시간 경과에 따른 게임 진행 관리
- 게임 타이머 (30분 기준)
- 시간대별 적 종류 및 수량 변경 (난이도 곡선)
- 보스 스폰 타이밍
- 스테이지별 이벤트 (보물상자, 특수 적 등)

의존: EnemySystem (적 스폰 제어), Game (타이머)
```

## 데이터 흐름

### 전투 사이클
```
Player 이동
    ↓
WeaponSystem: 쿨타임 체크 → 투사체 발사
    ↓
투사체 이동 → 적과 충돌 판정
    ↓                    ↓
  [적중]              [빗나감]
    ↓                    ↓
적 HP 감소            투사체 소멸
    ↓
  [적 사망?]
   Yes → Enemy.onDeath(game)
    ↓     (킬수++, 골드, 보석 드롭, 보스상자)
    ↓     → 플레이어 보석 흡수 → 경험치 +
    ↓                                ↓
                                [레벨업?]
                                    Yes → LEVELUP 상태
                                      ↓
                                  무기/패시브 선택
                                      ↓
                                  PLAY 상태로 복귀
```

### 피격 사이클
```
적이 플레이어에게 접촉
    ↓
플레이어 HP 감소 + 넉백 + 무적시간 시작
    ↓
  [HP <= 0?]
   Yes → GAMEOVER 상태
   No  → 무적시간 동안 깜빡이는 이펙트
```

```
(아래부터 위로 그린다 = 아래가 먼저 그려짐)

Layer 0: 배경 (타일맵 or 단색)

Layer 2: 적 (Enemies)
Layer 3: 플레이어 (Player)

Layer 6: HUD (체력바, 경험치바, 타이머)
Layer 7: UI 오버레이 (레벨업 선택, 일시정지, 게임오버)
```

## 오브젝트 풀링 전략

> "왜 필요한가?" - 적이 100마리, 투사체가 50개, 보석이 200개...
> 매번 new로 생성하고 삭제하면 성능이 떨어진다.
> 미리 만들어두고 재활용하는 것이 오브젝트 풀링.

```
풀링 대상:
├── Enemy Pool       (최대 250개 미리 생성)
├── Projectile Pool  (최대 150개 미리 생성)
├── Gem Pool         (최대 400개 미리 생성)
├── DamageText Pool  (최대 30개 미리 생성)
└── Chest Pool       (최대 5개 미리 생성)

동작 방식:
1. 게임 시작 시 → 풀에 비활성 객체를 미리 생성
2. 적 스폰 시 → 풀에서 비활성 적을 꺼내 활성화 + 위치/스탯 설정
3. 적 사망 시 → 삭제하지 않고 비활성화 → 풀로 반환
4. 풀이 다 찼는데 더 필요하면 → 풀 크기 확장 (경고 로그)
```

## 파일 의존성 맵

```
main.js
└── Game.js
    ├── Input.js            (키보드/마우스 입력)
    ├── Camera.js           (카메라/뷰포트)
    ├── DebugMode.js        (디버그 도구: F1 토글)
    ├── Player.js           (이동, 체력, 무기 장착)
    ├── entities/
    │   ├── Enemy.js        (적 개체, onDeath 통합)
    │   ├── Gem.js          (경험치 보석)
    │   ├── Projectile.js   (투사체)
    │   ├── DamageText.js   (데미지 숫자)
    │   └── Chest.js        (보물상자)
    ├── weapons/            (기본 8종 + 진화 8종)
    ├── systems/
    │   ├── EnemySpawner.js (웨이브/보스 스폰)
    │   ├── CollisionSystem.js (충돌 판정)
    │   ├── ExpSystem.js    (레벨업 선택지)
    │   ├── EvolutionSystem.js (무기 진화)
    │   └── UpgradeSystem.js (영구 업그레이드)
    ├── ui/
    │   ├── HUD.js, MenuUI.js, CharSelectUI.js
    │   ├── LevelUpUI.js, EvolutionUI.js
    │   ├── PauseUI.js, UpgradeShopUI.js
    │   ├── GameOverUI.js, VictoryUI.js
    │   └── (모든 UI는 Game 인스턴스 참조)
    ├── data/
    │   └── config.js       (모든 게임 수치 통합)
    └── utils/
        ├── MathUtils.js, ObjectPool.js
        ├── ErrorGuard.js, Storage.js
```
